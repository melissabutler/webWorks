How Web Works Exercise

Part One: Solidify Terminology

  1. What is HTTP?

    2. HTTP is an abbreviation for Hypertext Transfer Protocol, which in layman's terms, is the way in which a browser communicates with a server.

  2. What is a URL?

    3. A URL is a string of information that acts as the "address" for a given resource.

  3. What is DNS?

    4. DNS is the abbreviation for domain name service, which acts as the "phonebook" of the internet. It translates between human readable information, such as website names, to computer readable information, or IP addresses.

  4. What is a query string?

    5. A query string is a string added onto a website's URL, that pass specific information into the web browser, and change the user experience.

  5. What are two HTTP verbs and how are they different?

    6. Two HTTP verbs are GET and POST.

    GET requests information from the source, and retrieves said data, without changing the server's data.

    POST submits information to the source, and will change the server's data in some way.

  6. What is an HTTP request?

    7. An HTTP request is typically a GET request, in which your browser is asking the server for specific information, usually in the form of a URL.

  7. What is an HTTP response?

    8. An HTTP response is the reaction to a request, the server sending back the requested date, for example, the HTML text for the given URL webpage.

  8. What is an HTTP header? Give a couple examples of request and response headers you have seen.

    9. A header contains the information of a request. This will include metadata such as language, the date, the format of the data, etc.

  9. What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser?

    10. When one types “http://somesite.com/some/page.html” into the browser, the browser sends a GET request to the server. If the server has the information related to said URL, it will send said information back, rendering the data as a webpage. The browser will send multiple requests, depending on the contents of the request, including, for example, the html code, as well as linked css, javascript, and any other relevant links such as images.
